# Handoff: FreeNewsApi Node API

## Project State

A TypeScript + Express API that syncs news from FreeNewsApi.io into SQLite every hour. Serves a mobile news-reading app. Complete and running.

**Location:** `C:\Users\Diogo Alves\Desktop\Codigo\app-sistemasmoveis\api`

## What's Built

### Core API
- Hourly sync from FreeNewsApi.io (9 categories, pt-419/BR, 3 pages/category)
- `GET /news` — paginated listing from local DB, filterable by `?category=`
- `GET /details?uuid=` — cache-aside pattern: fetches from FreeNewsApi on first access, persists to SQLite, returns full article body
- `GET /health`

### Auth
- `POST /auth/register` — email, username, password → returns JWT (bcrypt 12 rounds, HS256, 7d expiry)
- `POST /auth/login` — by email or username
- JWT auth middleware at `src/middleware/auth.ts`

### User Features (all under `/me/*`, require `Authorization: Bearer <token>`)
- `GET /me/news` — same as `/news` but filtered by user's muted keywords
- `POST /me/mutes` — add keyword `{ "keyword": "word" }`
- `GET /me/mutes` — list muted keywords
- `DELETE /me/mutes/:id` — remove mute
- `POST /me/read-later` — save article `{ "articleUuid": "..." }`
- `GET /me/read-later` — list queue (48h auto-expiry via query filter)
- `DELETE /me/read-later/:articleUuid` — remove from queue

### Tech Stack
- Express + TypeScript (ESM, ES2022)
- better-sqlite3 + Drizzle ORM
- axios (FreeNewsApi client), zod (validation), bcrypt (password), jsonwebtoken
- tsx (dev runner)

## Architecture

```
src/
  index.ts              — Express bootstrap + hourly cron
  db/
    schema.ts           — 5 tables: articles, article_categories, users, user_muted_keywords, user_read_later
    index.ts            — DB connection (WAL mode, foreign keys)
  lib/
    response.ts         — sendSuccess / sendError
    validate.ts         — Zod validation wrapper + ValidationError
    paginate.ts         — PaginationMeta builder
  services/
    fetcher.ts          — FreeNewsApi HTTP client
    article.ts          — listArticles (with keyword filtering) + getArticleDetails (cache-aside)
    sync.ts             — Hourly sync per category
    auth.ts             — register + login
    muting.ts           — add/list/remove muted keywords
    readLater.ts        — save/list/remove queue (48h expiry)
  routes/
    news.ts, details.ts, auth.ts, me.ts
  middleware/
    error.ts            — Global error handler
    auth.ts             — JWT verification
```

## Key Design Decisions
- **Cache-aside for article details**: sync only fetches lightweight listings; full body fetched on first user request, then cached
- **Keyword muting filters on title + publisher** (body isn't always loaded)
- **48h expiry** is a `>=` filter on `saved_at` — no cron needed
- **Sync is bounded**: max 3 pages (150 articles) per category, 1.1s delay between requests

## Run
`npm run dev` — starts server on port 3000, runs initial sync, then every hour
